# workspace/ — 你实际干活的地方

这里放你执行学习计划时**真正产出的东西**，让"做了什么"沉淀成文件，而不是停在聊天里。
`.skill-state.json` 只记进度索引，具体产出物的正文都在这。

## 怎么组织（按需，不强求）

```
workspace/
├── day-01/        每天的产出：提示词、截图、笔记、报错、代码片段……当天做啥就放啥
├── day-02/
├── ...
├── portfolio/     里程碑级别的作品集成品（能放简历、能演示的 demo + 说明）
└── notes.md       跨天的学习笔记 / 踩过的坑 / 知识点
```

- **day-NN/** 由 `career-ai-transition-status` 在你打卡当天按需创建，形态不限——这一天产出是提示词就放提示词，是 Python 脚本就放脚本，是文献笔记就放 md。
- **portfolio/** 只放成型的作品：每件一个子目录，含成品本身 + 一页 `说明.md`（输入/流程/输出/适用场景/价值）。
- **notes.md** 放不属于某一天、但要长期记住的东西。

## 和记忆的关系

- 你在这里产出文件 → `career-ai-transition-status` 打卡时把**文件路径**写进 `.skill-state.json` 的 `completed_tasks`。
- 所以换会话 / 换设备 / 换 AI，打开这个目录就能完整复现"哪天做了什么、东西在哪"。
- 复盘（`lessons.md`）回看时，直接顺着这些路径找到当时的产出。
